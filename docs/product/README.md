# Product Docs

Product docs preserve the product reasoning behind OpenIM work.

Use this directory for:

- PRDs: `docs/product/requirements/REQ-0000-short-name.md`
- UX/UI designs: `docs/product/designs/REQ-0000-short-name.md`

GitHub Issues should link to these docs. Issues track discussion and status; these files preserve decisions.

## PRD Minimum Contents

- Problem.
- Target users.
- Goal.
- Non-goals.
- User stories or requirements.
- Acceptance criteria.
- Risks.
- Open questions.

## UX/UI Minimum Contents

- User flow.
- Page or component states.
- Empty, loading, error, and disabled states.
- Interaction details.
- Accessibility notes.
