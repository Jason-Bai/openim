# 三、默认 BOT 完整命令逻辑（可直接开发）

每个 OpenClaw **默认拥有一个系统 BOT**
用户在聊天框输入 `/命令`，默认 BOT 直接回复。

---

## 1. `/help`

返回：

```text
你好！我是你的默认助手 BOT
支持命令：
/help          查看帮助
/my-bots       查看你创建的BOT
/new-bot       创建新BOT
/delete-bot    删除BOT
/connect       连接BOT
/disconnect    断开BOT
```

---

## 2. `/my-bots`

逻辑：查询 `bots` 表，返回当前用户的 BOT 列表
返回格式：

```text
你的BOT列表：
ID: 101 | 名称: 客服Bot | 状态: 已连接
ID: 102 | 名称: 通知Bot | 状态: 未连接
```

---

## 3. `/new-bot`

返回：**外部 BOT 接入规范文档（固定文本）**

```text
新BOT接入指南：
1. 下载官方Plugin插件
2. 在Plugin中配置你的OpenClaw账号
3. 使用/connect获取连接凭证
4. 启动BOT即可自动连接
```

---

## 4. `/delete-bot {bot_id}`

逻辑：

- 检查 BOT 是否属于当前用户
- 从 `bots` 表删除

返回：

```text
BOT {id} 已删除
```

---

## 5. `/connect`（最核心）

### 流程

1. 返回用户的 BOT 列表
2. 让用户选择 ID
3. 输出 **连接信息（含 Plugin 配置）**

### 返回示例

```text
请选择要连接的BOT ID：
101 客服Bot
102 通知Bot

你选择了：101
=== 复制以下内容给你的BOT ===
BOT_ID: 101
USER: zhangsan
TOKEN: 5f2dac80e1f2
HOST: ws://im.公司.com
PORT: 8080
Plugin: official-bot-plugin.js
=== 复制结束 ===
```

### 你提到的 Plugin

Plugin = **系统官方提供的标准连接插件**
作用：

- 建立 WebSocket
- 鉴权 TOKEN
- 接收消息
- 发送消息
- 保持连接

**外部 BOT 只要引用这个 Plugin，填入上面信息，就能接入系统。**

---

## 6. `/disconnect`

流程：

1. 返回**已连接 BOT 列表**
2. 让用户选择 ID
3. 确认是否断开
4. 更新 `bots` 表状态

返回：

```text
已连接BOT：
101 客服Bot

请输入要断开的ID：101
确认断开？(y/n)
> y
BOT 101 已断开连接
```
