# Engineering Docs

Engineering docs preserve the technical reasoning and implementation plan behind OpenIM work.

Use this directory for:

- Technical designs: `docs/engineering/designs/REQ-0000-short-name.md`
- Implementation plans: `docs/engineering/plans/REQ-0000-short-name.md`

GitHub Issues and Pull Requests should link to these docs.

## Technical Design Minimum Contents

- Architecture impact.
- Files/modules affected.
- API, schema, protocol, or state changes.
- Data flow.
- Error handling.
- Test strategy.
- Rollout and rollback notes when relevant.

## Implementation Plan Minimum Contents

- Small verifiable tasks.
- Files to create or modify.
- Tests to write or update.
- Commands to run.
- Expected command outcomes.
